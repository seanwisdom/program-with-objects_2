Important Notes

Lesson 1.1

How the Internet Works

The internet is a collection of servers that communicate via the protocol HTTP.  The internet is accessed from a personal computer through a browser.

HTML

HTML is short for Hypertext Mark Up Language and is the language most commonly used to write code for web pages. The key component of HTML is an element. An element is opened and closed by a tag. Elements are classified as either inline or block. A tag can also have an attribute. For example, the anchor tag uses the href attribute to set the target of the anchor. Following this link will take you to a search engine that you could use to search for other examples of attributes.

Lesson 1.2

Blocks

A web page is composed of Blocks. Block elements for a "box" around their content.

Tree Structure

HTML has what is a called a tree structure because elements can have other elements inside them. This creates a series of nested boxes. When writing HTML code the indents represent the nesting structure of the boxes.

Text Editors

Text Editors are used to write code. Text editors have features that make programming easier. For example, they can automatically give you options for different tags as you type or create closing tags for a new opening tag.

Lesson 1.3

Cascading Style Sheets

Cascading Style Sheets (CSS) add color, design, etc. to your web page. One of the key concepts in writing code is to limit how often you need to repeat yourself. You can do this by assigning classes to elements and then styling those classes instead of styling each element. This reduces the amount of code you need to write (saving time and potential mistakes) and makes changing styles a lot faster and easier. Elements can also inherit style from their "parents" in the DOM.

There are 3 ways to add style to your webpage.

  • Write the CSS in the head element of your HTML code.
  • Link your HTML code to a seperate CSS file.
  • Write your style in line with your HTML

Box Structure & Sizing

A box consists of the margin, border, padding, content. The margin is the transparent space between boxes and isn't calculated in the box sizing. Box sizing can be set as a specific number of pixels or a percentage of the available pixels.

Programming Process

The programming process consists of:

  1. Look for natural boxes
  2. Look for repeated styles and semantic headings
  3. Write your HTML
  4. Apply styles from largest to smallest
  5. Fix things

Repeat steps 3 & 4 as needed!

Lesson 2.1

Computer Programming

Computers are machines that can run programs. Programs tell the computer what to do. They do this through a variety of programming languages. Computers can run these programs very quickly.

Python

Python is an example of a programming language. Python is an interpreter that translates the high level code entered into machine language. The grammar of must be exact in order for the computer to understand it. A computer can't interpret imprecise grammar the way a person can.

Expressions & Operators

An expression is a value (number, string, etc.). An operator tells the program what action to take (arithmetic, comparison, assignment, etc).

Lesson 2.2

Variables

Variables allow you to give names to values. You assign a value to a variable using the = sign. The = sign means something completely different in programming versus arithmetic. You can think of the = sign as meaning "is" instead of "is the same as". The value of a variable can also be changed. Variables can be very useful in programming. They allow you to:

  • Make it easier to read your code by assigning names people can understand
  • Store the value of data to be used later in your code
  • Change the value of something

Strings

A string is a sequence of characters surrounded by an ', '' or '''. These characters can be any key on your keyboard. When working with variables and strings, the + has a different meaning than it does when used in arithmetic. It means to concatenate variables instead of adding them together. For example, "2" + "2" = 22, not 4.

The + Symbol

When working with variables and strings, the + symbol also has a different meaning than it does when used in arithmetic. It means to concatenate instead of add. For example, "2" + "2" = 22, not 4.

Lesson 2.3

Functions

Functions take an input, does something with that input and returns an output. Functions are very valuable in programming because they allow you to avoid repetition. Once a function is defined once it is defined forever in the program and can be used repetitively.

Structure of a Procedure

The first step in writing a procedure is to define the procedure with def. You will also need to have a return. This tells the computer what to return when the procedure is complete. Without this the procedure will return none or no value.

Lesson 2.4

If Statements

If statements execute a block of code when a test expression is evaluated to be true. The else statement tells the program what to do if the test condition is false. You don't always need to use an else statement, because when the if statement evaluates to false the program automatically skips the block of code inside of the if statement and moves on to the next line of code outside of the if statement. Sometimes the else statement is used to make the code easier to understand and read.

While Loops

A while loop will allow you to run a certain block of code continuously until a certain condition is met. Sometimes you'll want to insert a break into your loop. This can quickly take you out of the loop if a certain condition is met within the loop using an if statement.

Lesson 2.5

Debugging

The most important point in this section is to remember that "bugs" happen! It's part of programming and is unavoidable. As a programmer you need to have a structured, scientific approach to finding and fixing bugs. A good process to remember is:

  1. Examine error messages
  2. Work from example code
  3. Make sure examples work
  4. Check intermediate results
    • Add print statements to print variables within the code
  5. Keep and compare old versions
    • This could be as simple as commenting out your old code or using version control software

Lesson 2.6

Lists

Lists are similar to a string. While a string is a way to store a sequence of characters a list is a method of storing a sequence of anything (e.g. strings, numbers or other lists). You can use an index on a list to find an item(s) within the list. A list that's stored within another list is called a nested list.

Mutation and Append

Mutation refers to changing values within a list. Any value within the list can be changed by finding that value and then replacing it with a new value. You can also add new elements to the end of a list using the append function. This changes the existing list. You can also add two lists together to create a new list using the + operator.

Aliasing

Aliasing refers to having multiple ways to refer to the same object. If multiple variables refer to the same value then changes in that value affect both variables.

For Loops

A for loop is used to create a loop on a a list. It creates a variable that references each item in the list and then performs a block of code on that variable.

Lesson 2.7

Understanding Problems

Writing code is about solving problems. A problem can be broken down into:

  • Possible inputs
  • Desired outputs
  • Relationship between the inputs and outputs

Solving Problems

The following process will be very helpful when working through problems.

  1. What are the inputs
  2. What are the outputs
  3. Consider how a human would solve and develop a simple mechanical solution
  4. Write pseudocode to gain a better understanding of what the actual code will look like
  5. Develop incrementally and test as you go

Lesson 3

Object Oriented Programming

Objected Oriented Programming (OOP) is a programming language model based on identifying objects and how they relate to each other and interact. Objects can be things like people, pets, buttons, navigation bars, etc. The objects are the primary focus of designing the program. These objects each have their own role in the program and interact with each other to perform the functions that the program is intended to execute. Each object is given attributes and has methods associated with it. The methods are the logic that tell the object what to do within the program.

Introduction to Abstraction

Objected Oriented Programming utilizes a key concept called abstraction to create models for programming. Abstraction means to utilize a module of code without knowing all of the details about how the code you're utilizing is working. This makes it easier to write more complex code, because you don't have to figure out every detail of the code you're writing. You can utilize code created and packaged by other programmers to perform functions within your program.

Libraries and Modules

There are two types of libraries used in Python. The Python Standard Libraries that come packaged with Python. And External Libraries like the Twillio libary we used in one of the project examples we worked on. External libraries are developed independently and do not come packaged with Python. Libraries are a collection of modules typically organized around a theme. A module is a file containing Python definitions and statements. Modules can be called into python code very similarly to how external files can be called into HTML code.

Using Classes

In this section we created several programs that introduced us to utilizing classes. A class is like a blueprint of a building and has certain pre-defined attributes. There can be multiple instances (objects) of a class. This instance would be like a specific house built utilizing the blueprint of the class. This is similar to how classes are used in HTML and CSS to group certain elements together and style them consistently. A change to the class in either will make changes to all of the objects or elements utilizing that class.

Making Classes

In this section we learned how to create our own classes and how to use inheritance to make programming easier and faster. Inheritance means that an instance can inherit methods or attributes from it's parent. The methods usually get or set information about the instance. You can also override a method inherited from a parent class by specifying that method in the instance and giving it a new value. Again, this is all very similar to HTML/CSS which utilizes inheritance as well.
